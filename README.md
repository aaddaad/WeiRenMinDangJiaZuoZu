# WeiRenMinDangJiaZuoZu
开源软件开发实践为人民当家做组小组作业
一、 学习目标
我们小组参与本课程的核心目标是：通过参与真实的开源项目，将开源理论转化为实践，成为一名合格的开源贡献者。 具体目标如下：
1.熟练运用 Git的精通分支管理、提交、合并、解决冲突等核心操作。熟练掌握 Fork-Pull Request 工作流，包括 Issue 创建、代码审查、CI/CD 集成等。
2.亲身体验并遵守开源社区的行为准则、贡献者协议和沟通礼仪，学会与全球开发者进行高效异步协作。
3.在真实的项目环境中，提升代码质量、文档撰写、测试编写和问题调试的能力，而不仅仅是完成作业。
4.养成经常开源阅读的习惯，浏览issue/PR讨论，记录学到的最佳实践笔记
二、 开源贡献计划
1.选择新手友好并且项目所使用语言与小组成员的技术方向相符的项目
2.从修复文档、更正错别字、优化注释或补充测试用例等低风险任务开始。目标是熟悉项目代码结构、构建流程和贡献规范，确保每位成员都能成功完成首次合并。
3.在技术熟练后主动认领和解决具体的功能增强或错误修复任务。在实现前，积极在相应的Issue中与项目维护者沟通实现方案，确保工作方向正确。认真对待每一次代码审查反馈，将其视为宝贵的学习机会，并迭代改进提交。
4.在项目熟悉后主动认领更具有挑战性的任务
